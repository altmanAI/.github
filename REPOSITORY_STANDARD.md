# AltmanAI Repository Standard v1.0

Every AltmanAI repository should communicate trust, ownership, and engineering quality.

## Required Files

- README.md
- LICENSE.md
- SECURITY.md
- CONTRIBUTING.md
- CODE_OF_CONDUCT.md
- CHANGELOG.md

## Engineering Expectations

Repositories should maintain:

- clear ownership;
- documented purpose;
- reproducible setup instructions;
- testing or validation evidence;
- dependency awareness;
- security review practices.

## AI Projects

AI repositories must document:

- intended capability;
- limitations;
- human review requirements;
- evaluation approach;
- privacy considerations;
- deployment status.

## Release Quality Gate

Before release:

1. Verify functionality.
2. Review security impact.
3. Document changes.
4. Validate user impact.
5. Obtain accountable approval where required.

AltmanAI GitHub is an accountability platform, not only a code archive.
