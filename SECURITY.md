# Security Policy

AltmanAI takes a security-first approach to everything we build. AI systems, decision-support tools, and productivity software only earn trust when they are secure, explainable, and keep people in control of their data.

## Reporting a vulnerability

If you discover a security vulnerability in any AltmanAI product or repository, please report it privately rather than opening a public issue.

**Contact:** security@altmanai.tech

Please include:

- A clear description of the vulnerability
- Steps to reproduce
- Potential impact
- Any relevant logs or screenshots (with sensitive data redacted)

## What not to include publicly

Do not post vulnerability details, proof-of-concept exploits, or affected user data in public GitHub issues, pull requests, or discussions. Send this information directly to the security contact above.

## Responsible disclosure

We ask that you:

- Give us a reasonable window to investigate and remediate before public disclosure — we aim to acknowledge reports and coordinate on a disclosure timeline in good faith.
- Do not test against production systems in ways that could degrade service, access other users' data, or cause data loss.
- Do not attempt denial-of-service, social engineering, or physical security testing against AltmanAI infrastructure or personnel.

## No exploitation rule

Security research is welcome. Exploitation of a vulnerability beyond what is necessary to demonstrate it — including accessing, modifying, or exfiltrating real user data — is not.

## Credentials and secrets

Never include API keys, tokens, passwords, or credentials in a GitHub issue, pull request, or public repository — including when reporting a security concern. Send them only through the private contact channel above, and only if directly relevant to remediation.

## Scope

This policy applies to all repositories in the AltmanAI GitHub organization and to the products they support, including DailyPilot and Sentinel Elite, unless a specific repository defines a narrower policy.
